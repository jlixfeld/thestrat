    Configuration Reference

    TheStrat uses Pydantic schemas for configuration validation, providing type safety, detailed error reporting, and comprehensive documentation.

    Overview

    Configuration in TheStrat follows a hierarchical structure:

    • FactoryConfig - Root configuration for complete pipeline setup
    • AggregationConfig - OHLCV timeframe aggregation settings
    • IndicatorsConfig - TheStrat indicator configurations
    • AssetClassConfig - Market-specific settings (crypto, equities, FX)

    Quick Start

    from thestrat import Factory

# Minimal configuration
config = {
    "aggregation": {
        "target_timeframes": ["5m", "1h"],
        "asset_class": "equities"
    },
    "indicators": {
        "timeframe_configs": [{
            "timeframes": ["all"],
            "swing_points": {"window": 5, "threshold": 2.0}
        }]
    }
}

pipeline = Factory.create_all(config)

    Schema Documentation

    TheStrat Schema Documentation

    Auto-generated documentation for all Pydantic configuration models.

    AssetClassConfig

    Configuration for a specific asset class with comprehensive market metadata.

    Field Type Default Description
    timezone str PydanticUndefined Timezone for market hours and timestamp handling
    session_start str PydanticUndefined Market session start time in HH:MM 24-hour format
    hour_boundary bool PydanticUndefined Whether to align aggregation to hour boundaries for hourly+ timeframes
    trading_hours str PydanticUndefined Human-readable description of trading hours

    Class Constants

    • REGISTRY: Registry of predefined configurations for each asset class
    • Available keys: crypto, equities, fx

    Configured Asset Classes

    crypto - Trading Hours: 24/7 continuous trading - Timezone: UTC - Session Start: 00:00 - Hour Boundary: True

    equities - Trading Hours: 9:30 AM - 4:00 PM ET (regular session) - Timezone: US/Eastern - Session Start: 09:30 - Hour Boundary: False

    fx - Trading Hours: Sunday 5 PM - Friday 5 PM ET (24/5) - Timezone: UTC - Session Start: 00:00 - Hour Boundary: True

    TimeframeConfig

    Configuration and validation for timeframes with comprehensive metadata.

    Field Type Default Description

    Class Constants

    • TIMEFRAME_TO_POLARS: Mapping of timeframe strings to Polars format strings
    • Available keys: 1min, 5min, 15min, 30min, 1h, 4h, 6h, 12h, 1d, 1w, 1m, 1q, 1y
    • TIMEFRAME_METADATA: Comprehensive metadata for each supported timeframe
    • Available keys: 1min, 5min, 15min, 30min, 1h, 4h, 6h, 12h, 1d, 1w, 1m, 1q, 1y

    Timeframe Details

    Timeframe Category Duration Description Typical Use
    1min sub-hourly 1m 1-minute bars for high-frequency analysis Scalping, entry timing, tick analysis
    5min sub-hourly 5m 5-minute bars for short-term patterns Day trading, quick reversals
    15min sub-hourly 15m 15-minute bars for intraday structure Swing entries, session analysis
    30min sub-hourly 30m 30-minute bars for session structure Half-hourly patterns, session transitions
    1h hourly 1h 1-hour bars for daily structure analysis Day trading context, hourly levels
    4h multi-hourly 4h 4-hour bars for swing trading Swing trading, multi-day holds
    6h multi-hourly 6h 6-hour bars for extended swing analysis Position trading context
    12h multi-hourly 12h 12-hour bars for daily session analysis Asian/Western session splits
    1d daily 1d Daily bars for trend analysis Trend identification, position trading
    1w weekly 1w Weekly bars for long-term structure Major trend analysis, portfolio allocation
    1m monthly 4w Monthly bars for macro analysis Long-term investing, macro trends
    1q quarterly 12w Quarterly bars for fundamental analysis Business cycle analysis, long-term allocation
    1y yearly 52w Yearly bars for multi-year analysis Multi-year trends, generational analysis

    SwingPointsConfig

    Configuration for swing point detection with comprehensive parameter documentation.

    Field Type Default Description
    window int 5 Number of bars for swing detection window (must be odd for symmetry)
    threshold float 5.0 Minimum percentage move to qualify as swing point

    GapDetectionConfig

    Configuration for gap detection with comprehensive threshold documentation.

    Field Type Default Description
    threshold float 0.001 Minimum gap size as decimal percentage (0.01 = 1% gap)

    TimeframeItemConfig

    Configuration for a single timeframe item with flexible timeframe targeting.

    Field Type Default Description
    timeframes list[str] PydanticUndefined List of timeframes to apply this configuration to
    swing_points UnionType[SwingPointsConfig, NoneType] None Optional swing point detection configuration for these timeframes
    gap_detection UnionType[GapDetectionConfig, NoneType] None Optional gap detection configuration for these timeframes

    IndicatorsConfig

    Complete configuration for indicators component with per-timeframe settings.

    Field Type Default Description
    timeframe_configs list[TimeframeItemConfig] PydanticUndefined List of timeframe-specific indicator configurations

    AggregationConfig

    Complete configuration for aggregation component with market-aware settings.

    Field Type Default Description
    target_timeframes list[str] PydanticUndefined List of target timeframes for OHLC aggregation
    asset_class Literal[crypto, equities, fx] "equities" Asset class determining market behavior and defaults
    timezone UnionType[str, NoneType] None Override timezone (None uses asset class default)
    hour_boundary UnionType[bool, NoneType] None Override hour boundary alignment (None uses asset class default)
    session_start UnionType[str, NoneType] None Override session start time HH:MM (None uses asset class default)

    FactoryConfig

    Root configuration for Factory.create_all method with complete pipeline setup.

    Field Type Default Description
    aggregation AggregationConfig PydanticUndefined Aggregation component configuration for OHLC timeframe conversion
    indicators IndicatorsConfig PydanticUndefined Indicators component configuration for TheStrat analysis
